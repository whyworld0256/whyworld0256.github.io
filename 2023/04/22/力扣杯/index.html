1.补给马车

题目描述

远征队即将开启未知的冒险之旅，不过在此之前，将对补给车队进行最后的检查。supplies[i] 表示编号为 i 的补给马车装载的物资数量。
考虑到车队过长容易被野兽偷袭，他们决定将车队的长度变为原来的一半（向下取整），计划为：

  • 找出车队中 物资之和最小 两辆 相邻 马车，将它们车辆的物资整合为一辆。若存在多组物资之和相同的马车，则取编号最小的两辆马车进行整合；
  • 重复上述操作直到车队长度符合要求。

请返回车队长度符合要求后，物资的分布情况。

解答

直接模拟完事儿了。每次求出最小的那组马车，返回更新后的数组。这样的操作重复len(supplies)-len(supplies)//2 次即可。

2.探险营地

题目描述

探险家小扣的行动轨迹，都将保存在记录仪中。expeditions[i] 表示小扣第 i 次探险记录，用一个字符串数组表示。其中的每个「营地」由大小写字母组成，通过子串 -> 连接。

例：”Leet->code->Campsite”，表示到访了 “Leet”、”code”、”Campsite” 三个营地。

expeditions[0] 包含了初始小扣已知的所有营地；对于之后的第 i 次探险(即 expeditions[i] 且 i > 0)，如果记录中包含了之前均没出现的营地，则表示小扣 新发现 的营地。

请你找出小扣发现新营地最多且索引最小的那次探险，并返回对应的记录索引。如果所有探险记录都没有发现新的营地，返回 -1

注意：

  • 大小写不同的营地视为不同的营地；
  • 营地的名称长度均大于 0。

解答

先用split 拆出到达的地点，用哈希表来统计即可，注意特判情况。

3.最强祝福力场

题目描述

小扣在探索丛林的过程中，无意间发现了传说中“落寞的黄金之都”。而在这片建筑废墟的地带中，小扣使用探测仪监测到了存在某种带有「祝福」效果的力场。
经过不断的勘测记录，小扣将所有力场的分布都记录了下来。forceField[i] = [x,y,side] 表示第 i 片力场将覆盖以坐标 (x,y) 为中心，边长为 side 的正方形区域。

若任意一点的 力场强度 等于覆盖该点的力场数量，请求出在这片地带中 力场强度 最强处的 力场强度。

注意：

  • 力场范围的边缘同样被力场覆盖。

提示：

  • 1 <= forceField.length <= 100
  • forceField[i].length == 3
  • 0 <= forceField[i][0], forceField[i][1] <= 10^9
  • 1 <= forceField[i][2] <= 10^9

解答

我们想象现在有一大片网格点，当出现一块力场时，我们就给力场覆盖范围里的点加1，最终就是要去求整片网格中点的最大值。

现在有几个难点：

  • 由于力场的数量远小于值域，这片网格点该怎么表示
  • 有可能出现坐标为0.5的点，该怎么解决
  • 如何给网格覆盖范围中的点都加1

解决方案

  • 离散化
  • 坐标*2
  • 二维差分

预处理

离散化+差分

4.传送卷轴

题目描述

题意比较复杂，请点击链接查看。

题解

题目中有一句话：假设小扣和守护者都按最优策略行事, 然后我们需要去求 返回小扣需要在[附加负面效果]的情况下最少移动多少次才能到达魔法水晶。

翻译题目如下：

小扣有很多条从起点通往终点的路径，在一条给定路径上，守护者有很多个地点可以选择使用传送卷轴，这样会得到一个$k$ ，其表示在当前这条路径上的若干选择中，小扣被附加负面效果后移动次数的最大值，然后对于每条路径，小扣想选择一条路径来使$k$达到最小 ，因此问题的本质就是要我们去求$\min \max k$。

即最小化最大值，因此我们考虑二分答案。

首先我们先预处理出终点到其他每个节点的最短距离。

二分的关键在于$check$ 函数怎么写。$check$ 函数的作用在于如果现在给了我们一个$max_dis$ ,我们需要去判断能否在施加了负面效果下，小扣移动小于等于$max_dis$ 步可以走到终点。

那么我们就去搜索（写一个$dfs$ 函数）从起点到终点的所有路径，只要这条路径上存在一个空地，守护者在这里使用卷轴后使小扣在被施加负面效果后的移动距离大于$max_ dis$ 那么$dfs$ 就返回false ，反之如果在某条路径上，守护者在任何能使用卷轴的地方都使用了卷轴，但小扣在施加了负面效果后的移动距离总是小于$max_ dis$ 那么$check$ 便返回true。

step1: 找到起点和终点

step2： 预处理出从终点到其余所有点的最短路距离，如果无法到达则初始化为inf

step3： 剪枝，如果从起点都无法到达，直接返回-1

step4：check 函数
