统计公平数对的数目

题目描述

你一个下标从 0 开始、长度为 n 的整数数组 nums ，和两个整数 lower 和 upper ，返回 公平数对的数目 。

如果 (i, j) 数对满足以下情况，则认为它是一个 公平数对 ：

  • 0 <= i < j < n
  • lower <= nums[i] + nums[j] <= upper

分析

显然，我们将数组排序后再去考虑这个问题与原问题是一样的。

对于一个固定的nums[i],则 nums[j] 的范围为: $[lower-nums[i],upper-nums[i]]$

也就是说，我们只需要去求数组中，范围在$[lower-nums[i],upper-nums[i]]$ 中的数的个数$k$。

然后我们有：$k=$ 小于等于$upper-nums[i]$ 的数的个数 $-$ 小于 $lower-nums[i]$ 的数的个数

  • 小于等于$upper-nums[i]$ 的数的个数 $=$ 第一个大于$upper-nums[i]$ 的数的下标
  • 小于$lower-nums[i]$ 的数的个数 $=$ 第一个大于$lower-nums[i]$ 的数的下标

注意：为了避免重复计算，上述所有所有的数均在范围$[0,i-1]$ 中去找。

  • 1 <= i <= n

n的范围为1000

分析

由于n的范围很小，我们可以考虑预处理出每个数的惩罚数是多少，这个问题可以转化为前缀和。

找到最长的半重复子字符串

题目描述

给你一个下标从 0 开始的字符串 s ，这个字符串只包含 0 到 9 的数字字符。

如果一个字符串 t 中至多有一对相邻字符是相等的，那么称这个字符串 t 是 半重复的 。例如，0010 、002020 、0123 、2002 和 54944 是半重复字符串，而 00101022 和 1101234883 不是。

请你返回 s 中最长 半重复 子字符串的长度。一个 子字符串 是一个字符串中一段连续 非空 的字符。

分析

题目要求一个字符串中最长的半重复子字符串的长度，很自然的想法是考虑双指针。

我们的右指针不断向前走，并且不断判断当前子字符串是否是半重复的，这里可以去维护当前子字符串中相邻并相同字符对的个数

只要这样的字符对个数为2时，说明当前子字符串不合法，我们便去移动左指针，移动多少呢？

我们不断移动左指针，使得当前左指针所指位置和其后一个位置的元素相同即可，这样我们可以将该子字符串中的一个相邻并相同字符对去掉。
